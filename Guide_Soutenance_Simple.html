3. Conseils par Categorie

3.1 UML

Conseil Principal: Traitez les diagrammes UML comme outils de communication. Le jury evalue votre capacite a expliquer et justifier vos choix de conception.

  • Toujours inclure acteur "visiteur" quand pertinent
  • Utiliser "include" pour comportement obligatoire, "extend" pour optionnel
  • NE PAS mettre clefs etrangeres comme attributs
  • Creer classes meres abstraites quand pertinent
  • Utiliser tables pivots pour relations many-to-many

3.2 SQL

Conseil Principal: SQL n'est pas une question de memorisation - c'est la comprehension des relations de donnees.

  • INNER JOIN: lignes correspondantes des deux tables
  • LEFT JOIN: toutes les lignes de gauche + correspondances
  • GROUP BY toujours avec fonctions d'agregation
  • HAVING pour filtrer apres aggregation (pas WHERE)
  • Pratiquer: YEAR(), MONTH(), DATE() pour manipulation de dates

3.3 JavaScript

Conseil Principal: Les problemes de logique testent vos capacites de resolution, pas votre memoire. Decomposez les problemes en etapes.

  • Passer 10-15 minutes avant de regarder la solution
  • Ecrire d'abord l'approche en langage clair
  • Pratiquer sans utiliser les methodes integrees (ex: sans reverse())
  • Ne pas dependre du compilateur pour detecter les erreurs

3.4 POO

Conseil Principal: POO c'est penser en termes d'objets et de relations, pas seulement de syntaxe.

  • public: accessible partout | protected: classe et sous-classes | private: classe uniquement
  • Classe abstraite: structure commune, ne peut etre instanciee directement
  • Interface: contrat sans implementation
  • Heritage: eviter duplication de code
  • Static: apparient a la classe, pas aux instances
  • $this: instance actuelle | self: classe actuelle | parent: classe parente

5. Exercices Supplementaires

JavaScript - 15 Exercices

Facile: Somme tableau, max/min, verifier pair, compter elements, premier element

Moyen: Inverser sans reverse(), mot le plus long, supprimer doublons, fusionner tries, palindrome

Difficile: Compression "aaabbb"->"a3b3", validation parentheses, matrice en spirale, paires somme=X, split()

SQL - 15 Requetes

Facile: SELECT actifs, COUNT produits, AVG prix, ORDER BY, LIKE

Moyen: JOIN 3 tables, GROUP BY COUNT, HAVING, sous-requete max, BETWEEN dates

Difficile: Sous-requete avec JOIN, moyenne mobile, ROW_NUMBER, requete recursive, transaction

6. Resume des Points Cles

  1. Logique Algorithmique est Cruciale - 70% echec JavaScript. Pratiquer quotidiennement.

  4. POO Requiert Comprehension - Savoir quand utiliser chaque concept.
  5. Pratiquer Sous Pression - Se chronometrer pour simuler examen.
